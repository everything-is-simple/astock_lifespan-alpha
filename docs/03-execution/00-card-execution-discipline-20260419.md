# 执行卡纪律

日期：`2026-04-19`
状态：`生效中`

## 1. 进入执行前

正式任务进入执行前，必须尽量具备：

1. 对应 design
2. 对应 spec
3. 已创建 card

如果只是探索或问题定位，可以暂不建卡；一旦进入正式施工，就必须补齐。

## 2. 执行中

执行卡至少要说明：

- 本次要解决什么问题
- 设计和规格输入是什么
- 任务切片是什么
- 范围内与范围外是什么
- 收口标准是什么

## 3. 执行后

执行结束不能只停在“代码已经改完”，还必须有：

1. evidence：命令、测试、关键结果、产物路径
2. record：实际施工过程、偏差、备注
3. conclusion：正式裁决与影响说明

## 4. 例外说明

阶段一基础重构属于一次性补记批次，因为代码已经在治理恢复前落地。

从下一阶段开始，不再把补记视为常规流程。
